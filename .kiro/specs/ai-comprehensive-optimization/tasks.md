# 实现计划：国际跳棋AI深度优化

## 概述

本实现计划将国际跳棋AI引擎的深度优化需求转化为可执行的编码任务。基于现有的C++17代码库（boyi/boyi.cpp），我们将按优先级实现9个核心优化模块。任务按照高、中、低优先级组织，确保核心功能优先实现，可选功能可根据时间灵活调整。

## 任务列表

### 阶段1：高优先级优化（核心功能）

#### 1. 完善静态搜索以解决水平线效应

- [x] 1.1 增强Quiescence Search的吃子搜索逻辑
  - 修改SearchEngine::quiescence_search方法
  - 实现stand_pat评估作为下界
  - 实现beta剪枝（stand_pat >= beta时立即返回）
  - 添加递归深度限制（最大10层）
  - 添加重复局面检测（返回和棋分数0）
  - _需求: 1.1, 1.2, 1.3, 1.5, 1.6_

- [x] 1.2 实现MVV-LVA吃子排序
  - 在quiescence_search中为吃子走法添加MVV-LVA排序
  - 吃王优先级高于吃兵
  - 使用较小价值的棋子吃较大价值的棋子优先
  - _需求: 1.4_

- [ ]* 1.3 编写静态搜索单元测试
  - 测试stand_pat剪枝正确性
  - 测试MVV-LVA排序正确性
  - 测试递归深度限制
  - 测试重复局面检测
  - _需求: 1.1-1.6_

- [ ]* 1.4 编写静态搜索属性测试
  - **属性1: Round-trip一致性**
  - **验证: 需求1.7**
  - 对于包含连续吃子序列的局面，解析后再打印后再解析应产生等价局面
  - _需求: 1.7_

#### 2. 优化着法排序以提升剪枝效率

- [x] 2.1 实现完整的着法排序优先级系统
  - 修改SearchEngine::sort_moves和score_move方法
  - 实现优先级顺序：置换表最佳着法 > 强制吃子 > 产生王棋 > 杀手走法 > 历史启发高分 > 安静走法
  - 为吃子走法实现MVV-LVA评分
  - 集成KillerMoves检查（当前深度的两个杀手走法）
  - 集成HistoryTable评分（使用get(from, to)）
  - _需求: 2.1, 2.2, 2.3, 2.4_

- [x] 2.2 实现中局着法排序增强
  - 检测游戏阶段（开局/中局/残局）
  - 在中局阶段额外奖励中心控制走法（格子21,22,26,27）
  - 在中局阶段额外奖励王的活跃度走法
  - _需求: 2.7_

- [x] 2.3 更新历史启发表和杀手走法
  - 在alpha_beta方法中，当发生beta剪枝且走法为安静走法时更新
  - 更新KillerMoves（存储当前深度的剪枝走法）
  - 更新HistoryTable（增加depth * depth分数）
  - _需求: 2.5, 2.6_

- [ ]* 2.4 编写着法排序单元测试
  - 测试优先级顺序正确性
  - 测试MVV-LVA评分
  - 测试杀手走法集成
  - 测试历史启发集成
  - 测试中局增强

#### 3. 改进位置评估函数

- [x] 3.1 实现Piece-Square Tables
  - 在Evaluator类中添加50元素的position_value数组
  - 初始化时根据格子到中心的距离计算价值（中心高，边缘低）
  - 使用曼哈顿距离或欧几里得距离计算到中心的距离

- [x] 3.2 集成Piece-Square Tables到位置评估

  - 普通兵使用基础值加前进程度奖励

- [x] 3.3 实现升王威胁和接近升王奖励

  - 距离 <= 2步：增加20分升王威胁奖励

- [ ] 3.4 实现边缘惩罚

### 阶段2：中优先级优化（重要增强）

#### 4. 实现空步裁剪算法

- [~] 4.1 在Alpha-Beta搜索中实现空步裁剪
  - 修改SearchEngine::alpha_beta方法
  - 当深度 >= 3且不在静态搜索中时尝试空步裁剪
  - 跳过当前回合，以reduced_depth = depth - 3搜索
  - 如果结果 >= beta，返回beta（剪枝成功）
  - _需求: 4.1, 4.2, 4.3_

- [~] 4.2 实现空步裁剪的禁用条件
  - 检测Zugzwang状态（只有王棋且王棋数 <= 3）
  - 检测上一步是否为空步（防止连续空步）
  - 检测残局阶段（总棋子数 <= 6）
  - 在这些情况下禁用空步裁剪
  - _需求: 4.4, 4.5, 4.6_

- [ ]* 4.3 编写空步裁剪单元测试
  - 测试空步裁剪触发条件
  - 测试剪枝成功情况
  - 测试Zugzwang禁用
  - 测试连续空步禁用
  - 测试残局禁用
  - _需求: 4.1-4.6_

#### 5. 增强底线保护和边缘安全评估

- [~] 5.1 实现底线保护评估
  - 在Evaluator类中添加evaluate_back_row方法
  - 检查黑方底线（格子0-4）是否有黑方棋子
  - 检查白方底线（格子45-49）是否有白方棋子
  - 底线有己方棋子：增加8分保护奖励
  - 底线为空且对手有升王威胁：减少15分暴露惩罚
  - _需求: 5.1, 5.2, 5.3, 5.4_

- [~] 5.2 实现边缘安全评估
  - 在Evaluator类中添加evaluate_edge_safety方法
  - 检测边缘格子（列0或列9）
  - 边缘有己方棋子且相邻格子有己方棋子：增加6分安全奖励
  - 边缘有己方棋子且相邻格子有对手棋子：减少10分威胁惩罚
  - _需求: 5.5, 5.6_

- [~] 5.3 集成底线和边缘评估到总评估
  - 修改Evaluator::evaluate方法
  - 调用evaluate_back_row和evaluate_edge_safety
  - 将结果累加到总评估分数
  - _需求: 5.1-5.6_

- [ ]* 5.4 编写底线和边缘评估单元测试
  - 测试底线保护奖励
  - 测试底线暴露惩罚
  - 测试边缘安全奖励
  - 测试边缘威胁惩罚
  - _需求: 5.1-5.6_

#### 6. 添加阵型评估

- [~] 6.1 实现棋子连结性评估
  - 在Evaluator类中添加evaluate_formation方法
  - 检查每个己方棋子的4个对角相邻格子
  - >= 2个对角相邻己方棋子：增加10分连结奖励
  - 0个对角相邻己方棋子：减少15分孤立惩罚
  - _需求: 6.1, 6.2, 6.3_

- [x] 6.2 实现兵链和双王配合评估
  - 检测3个或更多己方兵形成的对角线兵链：增加25分兵链奖励
  - 检测两个己方王在相邻对角线（距离4或6）：增加30分双王配合奖励
  - _需求: 6.4, 6.5_

- [~] 6.3 实现包围评估
  - 检测对手棋子是否被包围（4个方向中 >= 3个有己方棋子）
  - 包围成功：增加20分包围奖励
  - _需求: 6.6_

- [~] 6.4 集成阵型评估到总评估
  - 修改Evaluator::evaluate方法
  - 调用evaluate_formation
  - 将结果累加到总评估分数
  - _需求: 6.1-6.6_

- [ ]* 6.5 编写阵型评估单元测试
  - 测试连结性奖励和孤立惩罚
  - 测试兵链奖励
  - 测试双王配合奖励
  - 测试包围奖励
  - _需求: 6.1-6.6_

- [~] 6.6 Checkpoint - 验证中优先级优化
  - 确保所有测试通过，询问用户是否有问题

## 注意事项

- **标记为`*`的任务为可选任务**：这些主要是测试相关的子任务，可以根据时间和需求灵活跳过
- **每个任务都引用了具体的需求条款**：确保实现的可追溯性
- **Checkpoint任务**：在关键阶段设置检查点，确保增量验证
- **优先级顺序**：高优先级任务（1-3）必须完成，中优先级任务（4-6）重要但可延后，低优先级任务（7-8）可选
- **集成和验证**：最后阶段（9-10）确保所有优化协同工作并验证性能提升

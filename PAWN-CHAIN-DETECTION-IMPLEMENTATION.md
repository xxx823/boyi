## 功能说明

### 什么是兵链？

兵链（Pawn Chain）是指3个或更多己方兵在对角线上连续排列，形成防御或进攻阵型。这种阵型在国际跳棋中非常重要，因为：
- 提供相互保护
- 形成坚固的防线
- 创造进攻机会
- 控制关键区域

### 检测算法

#### 1. 对角线方向定义

国际跳棋棋盘上的对角线有4个方向：
- **左上**: 偏移量 -6
- **右上**: 偏移量 -4
- **左下**: 偏移量 +4
- **右下**: 偏移量 +6

#### 2. 检测流程

```
对于每个己方兵的位置：
    对于每个对角线方向：
        初始化链长度 = 1
        当前位置 = 起始兵位置
        
        循环：
            计算下一个位置 = 当前位置 + 对角线偏移
            
            如果下一个位置无效（超出棋盘）：
                跳出循环
            
            如果下一个位置没有己方兵：
                跳出循环
            
            验证行差 = 1（确保在同一对角线）：
                如果行差 != 1：
                    跳出循环
            
            链长度++
            当前位置 = 下一个位置
        
        如果链长度 >= 3：
            记录一条兵链
            跳出方向循环（每个兵只计数一次）
```

#### 3. 评分规则

- **兵链奖励**: 每条兵链 +25分
- **可累加**: 如果有多条兵链，分数累加

### 代码实现

```cpp
// 4. 检测对角线兵链（3个或更多己方兵形成对角线链）
int pawn_chain_count = 0;
uint64_t men_copy = my_men;
std::vector<int> men_positions;

// 收集所有兵的位置
while (men_copy) {
    int sq = __builtin_ctzll(men_copy);
    men_copy &= men_copy - 1;
    men_positions.push_back(sq);
}

// 检测对角线兵链
const int diagonal_offsets[] = {-6, -4, 4, 6};

for (int start_sq : men_positions) {
    for (int offset : diagonal_offsets) {
        int chain_length = 1;
        int current_sq = start_sq;
        
        // 沿着对角线方向检查连续的兵
        while (true) {
            int next_sq = current_sq + offset;
            
            // 检查下一个格子是否有效且有己方兵
            if (next_sq < 0 || next_sq >= 50) break;
            
            uint64_t next_mask = 1ULL << next_sq;
            if (!(my_men & next_mask)) break;
            
            // 检查是否在同一对角线上（行列关系正确）
            int curr_row = current_sq / 5;
            int next_row = next_sq / 5;
            int row_diff = abs(next_row - curr_row);
            
            // 对角线移动应该是行差1
            if (row_diff != 1) break;
            
            chain_length++;
            current_sq = next_sq;
        }
        
        // 如果形成3个或更多兵的链，计数
        if (chain_length >= 3) {
            pawn_chain_count++;
            break;  // 每个起始兵只计数一次
        }
    }
}

// 兵链奖励（每条链+25分）
if (pawn_chain_count > 0) {
    score += pawn_chain_count * 25;
}
```

## 与原实现的区别

### 原实现（连续攻击态势）

```cpp
// 检查是否在前进位置
if ((is_black && row >= 5) || (!is_black && row <= 4)) {
    advanced_count++;
}

// 如果有3个或更多棋子在前进位置
if (advanced_count >= 3) {
    score += 25;
}
```

**特点**：
- 只检查兵是否在前进位置
- 不要求兵在对角线上连续
- 评估整体进攻态势

### 新实现（对角线兵链）

**特点**：
- 严格检查兵是否在对角线上连续排列
- 验证行差为1（确保真正的对角线关系）
- 可以检测多条兵链
- 更精确的阵型评估

### 两者关系

新实现**保留了原实现**，两者互补：
- **兵链检测**：评估阵型质量（防御/进攻结构）
- **连续攻击态势**：评估整体进攻压力

## 性能考虑

### 时间复杂度

- 收集兵位置：O(n)，n为兵的数量
- 检测兵链：O(n × 4 × m)，m为平均链长度
- 总体：O(n)，因为m通常很小（<10）

### 优化建议

如果性能成为问题，可以考虑：
1. 使用位运算优化对角线检测
2. 缓存已检测的兵链
3. 限制最大检测长度

## 🔍 详细文档

### Bug 1: from字段错误
**文档**: `BUGFIX-CHECKERS-UI.md`

**问题**: 
- 递归生成连续跳吃时，走法的from字段指向中间位置而非原始起点
- 导致UI过滤走法时漏掉某些棋子

**修复**:
- 添加 `originalFrom` 参数追踪原始起点
- 确保所有走法的from字段都指向正确的起点

**影响**:
- 修复前：多个棋子能吃相同数量时，只有一个能移动
- 修复后：所有能吃最大数量的棋子都可以移动

---

### Bug 2: 路径交叉限制
**文档**: `BUGFIX-PATH-CROSSING.md`

**问题**:
- 代码错误地限制了落点不能重复
- 导致路径不能交叉，截断了长连跳
- 钻石阵地只能找到4连跳，漏掉第5个子

**修复**:
- 删除 `visitedSquares.has(landIndex)` 检查
- 允许路径自由交叉

**影响**:
- 修复前：钻石阵地只能吃4个子
- 修复后：能正确吃5个子，支持路径交叉

## 🔧 技术细节

### 修改位置

**文件**: `web/checkers.js`

1. **第197行**: 添加 `originalFrom` 参数
   ```javascript
   generateCaptureMoves(from, piece, row, col, capturedSoFar, visitedSquares, originalFrom)
   ```

2. **第247-250行**: 删除王棋落点重复检查
   ```javascript
   // 删除: || visitedSquares.has(landIndex)
   if (this.board[landIndex]) break;
   ```

3. **第253-265行**: 王棋递归传递 `originalFrom`
   ```javascript
   from: originalFrom,  // 使用原始起点
   ```

4. **第320-327行**: 删除普通棋子落点重复检查
   ```javascript
   // 删除整个 if (visitedSquares.has(landIndex)) 块
   ```

5. **第340-352行**: 普通棋子递归传递 `originalFrom`
   ```javascript
   from: originalFrom,  // 使用原始起点
   ```

### 关键概念

**屏障规则 vs 落点限制**:
- ✅ **屏障规则**: 不能跳过同一个被吃的棋子两次（正确）
- ❌ **落点限制**: 不能多次落在同一个空格（错误理解）

**正确实现**:
```javascript
// ✅ 检查被吃的棋子（屏障规则）
if (capturedSoFar.includes(enemyIndex)) continue;

// ✅ 检查落点是否为空
if (this.board[landIndex]) continue;

// ❌ 不要检查落点是否访问过
// if (visitedSquares.has(landIndex)) continue;  // 删除这行
```

# 国际跳棋UI Bug修复总结

## 修复的Bug列表

### Bug 1: 连续跳吃时from字段错误 ✅ 已修复
**文档**: `BUGFIX-CHECKERS-UI.md`

**问题**: 递归生成连续跳吃时，走法的from字段被设置为当前递归层的位置，而不是原始起点。

**修复**: 添加 `originalFrom` 参数追踪整个跳吃序列的起点。

**影响**: 
- 修复前：多个棋子能吃相同数量时，只有一个能移动
- 修复后：所有能吃最大数量的棋子都可以移动

---

### Bug 2: 错误限制落点不能重复 ✅ 已修复
**文档**: `BUGFIX-PATH-CROSSING.md`

**问题**: 代码错误地限制了落点不能重复，导致路径不能交叉，截断了长连跳。

**修复**: 删除 `visitedSquares.has(landIndex)` 检查，允许路径交叉。

**影响**:
- 修复前：钻石阵地只能找到4连跳
- 修复后：能正确找到5连跳，支持路径交叉

## 规则理解总结

### 国际跳棋连续跳吃规则

#### ✅ 正确理解
1. **屏障规则**: 不能跳过同一颗被吃掉的棋子两次
2. **路径自由**: 落点可以重复，路径可以交叉
3. **方向自由**: 吃子时可以向任意方向（包括向后）
4. **最大吃子**: 必须选择吃子最多的走法

#### ❌ 常见误解
1. ~~落点不能重复~~ → 错误！可以重复
2. ~~路径不能交叉~~ → 错误！可以交叉
3. ~~普通棋子不能向后吃~~ → 错误！吃子时可以向后

## 技术要点

### JavaScript递归最佳实践

#### ✅ 正确做法
```javascript
// 1. 使用展开运算符创建新数组
const newCaptured = [...capturedSoFar, enemyIndex];

// 2. 使用new Set创建新集合
const newVisited = new Set(visitedSquares);

// 3. 传递不可变参数
generateCaptureMoves(landIndex, piece, landRow, landCol, newCaptured, newVisited, originalFrom);
```

#### ❌ 常见错误
```javascript
// 1. 直接修改原数组（状态污染）
capturedSoFar.push(enemyIndex);
generateCaptureMoves(...);
// 忘记 pop()

// 2. 共享引用
const newCaptured = capturedSoFar;  // 错误！这只是引用
newCaptured.push(enemyIndex);       // 会修改原数组

// 3. 错误的深拷贝
const newCaptured = capturedSoFar;  // 浅拷贝，不够
```

### 规则实现要点

1. **屏障规则**: 使用 `capturedSoFar.includes(enemyIndex)` 检查
2. **落点检查**: 只检查 `board[landIndex]` 是否为空
3. **原始起点**: 使用 `originalFrom` 参数追踪
4. **状态隔离**: 每个递归分支使用独立的状态副本

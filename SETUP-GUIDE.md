## 🎯 系统概述

这是一个完整的国际跳棋人机对弈系统，包含：

- **Web前端**：美观的游戏界面（HTML+CSS+JavaScript）
- **Node.js后端**：API服务器，连接前端和AI引擎
- **C++ AI引擎**：强大的国际跳棋AI（Alpha-Beta搜索）

## 🔧 安装步骤

### 步骤1：检查系统要求

运行测试脚本检查系统：

```bash
双击 test-system.bat
```

需要确保：
- ✅ Windows 10/11
- ✅ Node.js 14+ 已安装
- ✅ C++引擎已编译（boyi.exe）

### 步骤2：安装Node.js（如果未安装）

1. 访问 https://nodejs.org/
2. 下载LTS版本（推荐 18.x 或 20.x）
3. 运行安装程序
4. 安装完成后，重启命令提示符

验证安装：
```bash
node --version
npm --version
```

### 步骤3：安装依赖包

**方法A：自动安装（推荐）**

直接运行 `start-all.bat`，首次运行会自动安装依赖。

**方法B：手动安装**

```bash
cd server
npm install
```

或双击 `server/install.bat`

### 步骤4：编译C++引擎（如果未编译）

1. 打开 `boyi.sln` 在Visual Studio 2022中
2. 选择配置：Debug, x64
3. 生成 → 重新生成解决方案
4. 确认生成成功：`x64/Debug/boyi.exe`

## 🚀 启动系统

### 方法1：一键启动（最简单）

```bash
双击 start-all.bat
```

这会：
1. ✅ 自动安装依赖（如果需要）
2. ✅ 启动后端服务器
3. ✅ 打开浏览器
4. ✅ 开始游戏！

### 方法2：分步启动

#### 启动后端服务器

```bash
cd server
node server.js
```

或双击 `server/start.bat`

看到以下信息表示成功：
```
========================================
  国际跳棋AI服务器已启动
========================================
  服务器地址: http://localhost:3000
========================================
```

#### 打开浏览器

访问：`http://localhost:3000`

### 方法3：仅前端（无AI）

如果只想测试界面，不需要AI：

```bash
cd web
双击 index.html
```

注意：这种方式只有简单的随机AI。

## 🎮 使用教程

### 第一次游戏

1. **启动系统**
   ```bash
   双击 start-all.bat
   ```

2. **等待加载**
   - 服务器窗口会打开（不要关闭）
   - 浏览器自动打开游戏界面

3. **开始游戏**
   - 点击"新游戏"按钮
   - 选择玩家颜色（黑方/白方）
   - 选择AI难度

4. **下棋**
   - 点击己方棋子（会高亮）
   - 点击黄色高亮格子移动
   - 等待AI走棋

### 功能说明

#### 游戏控制
- **新游戏**：重新开始
- **悔棋**：撤销上一步（开发中）
- **提示**：显示推荐走法

#### 游戏设置
- **玩家颜色**：
  - 黑方（先手）- 您先走
  - 白方（后手）- AI先走

- **AI难度**：
  - 简单（1秒）- 适合新手
  - 中等（3秒）- 适合练习
  - 困难（5秒）- 有挑战性
  - 专家（10秒）- 非常强

#### 游戏状态
- 当前回合：显示轮到谁
- 棋子数量：双方剩余棋子
- 走法数：已走步数

#### AI思考信息
- 搜索深度：AI搜索的层数
- 搜索节点：评估的局面数
- 评估分数：局面优劣（正数=黑方优势）
- 用时：AI思考时间

### 游戏规则

#### 基本规则
1. 黑方先手
2. 轮流走棋
3. 只能在黑格上移动

#### 移动规则
- **普通棋子**：只能向前斜走一格
- **王棋**：可以向任意斜方向移动

#### 吃子规则
- 跳过对方棋子进行吃子
- **强制吃子**：有吃子机会必须吃
- **最大吃子**：必须选择吃子最多的走法
- **连续跳吃**：一次可以吃多个棋子

#### 升王规则
- 黑方棋子到达第10行升王
- 白方棋子到达第1行升王
- 王棋用♔符号标记

#### 胜负判定
- 吃光对方所有棋子 → 获胜
- 对方无合法走法 → 获胜

## ❓ 常见问题

### Q1: 双击start-all.bat没反应？

**A:** 检查Node.js是否安装：
```bash
# 打开命令提示符
node --version

# 如果显示版本号，说明已安装
# 如果提示"不是内部或外部命令"，需要安装Node.js
```

### Q2: 浏览器显示"无法访问此网站"？

**A:** 服务器可能未启动成功：
1. 检查服务器窗口是否显示"服务器已启动"
2. 尝试访问 http://localhost:3000/api/health
3. 如果失败，查看服务器窗口的错误信息

### Q3: AI不走棋或走棋很慢？

**A:** 可能的原因：
1. **后端未连接**：检查服务器是否运行
2. **AI难度太高**：降低AI难度到"简单"
3. **系统降级**：系统会自动使用本地AI（较弱）

### Q4: 提示"C++引擎未启动"？

**A:** 这是正常的：
- 系统会自动使用JavaScript AI作为备用
- JavaScript AI较弱，但可以正常游戏
- 如需完整AI，确保 `x64/Debug/boyi.exe` 存在

### Q5: 如何重新编译C++引擎？

**A:** 
1. 打开 `boyi.sln` 在Visual Studio中
2. 确保配置为：Debug, x64
3. 生成 → 清理解决方案
4. 生成 → 重新生成解决方案
5. 检查 `x64/Debug/boyi.exe` 是否生成

### Q6: 端口3000被占用怎么办？

**A:** 修改端口：
1. 编辑 `server/server.js`
2. 找到 `const PORT = 3000;`
3. 改为其他端口，如 `const PORT = 8080;`
4. 重启服务器

### Q7: 如何查看详细错误信息？

**A:** 
1. 浏览器按F12打开开发者工具
2. 查看Console标签
3. 查看Network标签（API调用）
4. 服务器窗口也会显示错误

### Q8: 游戏卡住不动了？

**A:** 
1. 刷新浏览器页面（F5）
2. 点击"新游戏"重新开始
3. 如果仍然卡住，重启服务器

### Q9: 如何保存对局？

**A:** 
- 当前版本暂不支持保存
- 可以截图保存棋盘状态
- 走法历史会显示在右侧面板

### Q10: 可以两个人对战吗？

**A:** 
- 当前版本是人机对战
- 可以两人轮流操作同一台电脑
- 未来版本会支持双人对战

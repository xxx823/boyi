# 开局库手动测试指南

## 测试目标

验证开局库功能是否正常工作，包括：
1. ✅ 开局库文件加载成功
2. ✅ 初始局面能够命中开局库
3. ✅ 开局库返回合法走法
4. ✅ 统计信息正确输出
5. ✅ 查询速度符合要求（< 10微秒）

---

## 方法1：自动测试（推荐）

### 步骤1：编译程序

1. 打开 `boyi.sln`
2. 选择 **Debug** 或 **Release** 配置
3. 按 **F7** 编译
4. 确保编译成功，生成 `x64\Debug\boyi.exe` 或 `x64\Release\boyi.exe`

### 步骤2：运行测试脚本

双击运行 `test-opening-book.bat`

脚本会自动：
- 检查可执行文件是否存在
- 检查开局库文件是否存在
- 创建测试输入
- 运行程序并捕获输出
- 分析测试结果

### 步骤3：查看测试结果

测试脚本会显示：
- ✓ 开局库命中测试 - 通过/失败
- ✓ 统计信息输出 - 通过/失败
- ✓ 错误检查 - 无错误/发现错误

完整输出保存在 `test_output.txt`

## 方法2：手动测试

### 步骤1：编译程序

同上

### 步骤2：准备测试环境

确保以下文件在同一目录：
- `boyi.exe`（编译生成的可执行文件）
- `opening_book.txt`（开局库数据文件）

### 步骤3：运行程序

打开命令行，进入程序目录：

```bash
cd x64\Debug
```

运行程序：

```bash
boyi.exe
```

### 步骤4：输入测试命令

程序启动后，依次输入：

```
START
```

观察输出，应该看到：

```
开局库已初始化：X 个局面
从文件加载开局库：15 条开局线，156 个局面
```

然后输入：

```
MOVE
```

观察输出，应该看到：

```
INFO: Opening book hit!
MOVE: 6-11
TIME: 1
```

这表示开局库命中成功！

最后输入：

```
QUIT
```

程序结束时会输出统计信息：

```
========== 开局库统计 ==========
局面数量: 156
总查询次数: 1
命中次数: 1
命中率: 100%
估算节省时间: 3 秒
================================
```

## 预期结果

### 成功标准

1. **加载成功**：
   ```
   从文件加载开局库：15 条开局线，156 个局面
   ```

2. **命中成功**：
   ```
   INFO: Opening book hit!
   MOVE: 6-11
   TIME: 1
   ```

3. **统计正确**：
   ```
   命中率: 100%
   估算节省时间: 3 秒
   ```

### 失败情况处理

#### 情况1：找不到 opening_book.txt

**现象**：
```
开局库已初始化：5 个局面
```

**原因**：文件不存在或路径错误

**解决**：
1. 确认 `opening_book.txt` 在程序同一目录
2. 或者将文件复制到 `x64\Debug\` 目录

#### 情况2：开局库未命中

**现象**：
```
INFO: Searching...
MOVE: 6-11
TIME: 2500
```

**原因**：初始局面不在开局库中

**解决**：
1. 检查 `opening_book.txt` 是否包含初始局面的走法
2. 确认文件格式正确
3. 添加更多开局线

#### 情况3：程序崩溃

**现象**：程序异常退出

**原因**：可能是内存错误或走法解析错误

**解决**：
1. 使用 Debug 配置编译
2. 在 Visual Studio 中调试运行
3. 检查 `opening_book.txt` 格式是否正确

## 常见问题

### Q: 为什么初始局面没有命中开局库？

A: 检查以下几点：
1. `opening_book.txt` 是否在正确的目录
2. 文件是否包含初始局面的走法（如 `6-11 31-26 ...`）
3. 文件格式是否正确（走法用空格分隔）

### Q: 如何验证开局库的走法是合法的？

A: 
1. 使用 Web 界面手动验证
2. 在代码中添加走法合法性检查
3. 运行完整对局，观察是否有错误

### Q: 如何提高开局库命中率？

A:
1. 增加开局线数量
2. 增加每条开局线的深度（步数）
3. 覆盖更多常见开局变化

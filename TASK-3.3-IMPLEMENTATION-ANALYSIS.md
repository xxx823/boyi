## 逻辑正确性验证

### 1. 条件互斥性

```cpp
if (distance_to_promotion <= 2) {
    score += 20;  // 第一优先级
} else if (distance_to_promotion <= 4) {
    score += 10;  // 第二优先级
}
// 距离 > 4步：无奖励
```

✅ **正确**: 使用 `else if` 确保每个兵只获得一次奖励，且优先级正确（更近的威胁获得更高奖励）

### 2. 距离计算正确性

**黑方**:
- 从第1行（row=0）到第10行（row=9）
- 距离 = 9 - row ✓

**白方**:
- 从第10行（row=9）到第1行（row=0）
- 距离 = row ✓

### 3. 边界情况

**情况1**: 兵已经在升王线上（距离=0）
- 黑方：row=9 → 距离=0 → 不满足任何条件 → 无奖励 ✓
- 白方：row=0 → 距离=0 → 不满足任何条件 → 无奖励 ✓
- **正确**: 已升王的兵会变成王棋（black_kings/white_kings），不再在 black_men/white_men 中

**情况2**: 兵距离很远（距离>4）
- 不满足任何条件 → 无奖励 ✓
- **正确**: 符合需求，只奖励接近升王的兵

**情况3**: 兵恰好在边界（距离=2或4）
- 距离=2：满足第一个条件 → +20分 ✓
- 距离=4：满足第二个条件 → +10分 ✓
- **正确**: 使用 `<=` 确保边界情况被正确处理

## 对手威胁检测

```cpp
// 检查对手的升王威胁（需要阻止）
men_copy = opp_men;
while (men_copy) {
    int sq = __builtin_ctzll(men_copy);
    men_copy &= men_copy - 1;
    
    int row = sq / 5;
    int distance_to_promotion;
    
    if (!is_black) {
        distance_to_promotion = 9 - row;
    } else {
        distance_to_promotion = row;
    }
    
    if (distance_to_promotion <= 2) {
        score -= 15;  // 对手即将升王，需要阻止
    }
}
```

**逻辑验证**:
- 当前玩家是黑方（is_black=true）时，检查白方兵（opp_men）
  - 白方向下移动：distance = row ✓
- 当前玩家是白方（is_black=false）时，检查黑方兵（opp_men）
  - 黑方向上移动：distance = 9 - row ✓

**惩罚**: -15分（对手即将升王是危险的）
